## 📍 **따라하며 배우는 C언어**
> 원래는 인프런에 C++강의를 들을려고했는데 선수지식으로 C언어와 컴퓨터 구조강의 먼저 들어야한다해서 유튜브에있는 강의를 먼저 듣고 C++ 강의 결재해서 들을 예정이다.

### 📌 **C언어를 제대로 배우는 방법**
- C언어 기초문법 과 주변개념들을 공부 
- 따라하면서 배우기

### 📌 **0.0 초보 프로그래머를 위한 컴퓨터의 작동 원리 강의 개요**

**컴퓨터**
  - 하드웨어
  - 소프트웨어

### 📌 **0.1 컴퓨터의 구성 요소들**

- 입력장치 → 컴퓨터 본체 → 출력 장치

**컴퓨터 본체**
1. 메인 보드
2. 그래픽스 카드
   - Graphics Card
   - Graphics Processing Unit(GPU)
   - 자체적으로 펜이 들어가있음 즉 CPU도 들어가있음, 메모리도 들어가있음 
3. 디스플레이
4. 전원 공급장치
5. 중앙 처리장치
   - Central Processing Unit(CPU) 
6. 입출력 장치
7. 주 기억 장치 (Primary storage - Computer Memory)
   - Random-Access Memory (RAM) 
   - 전원을 끄면 데이터가 사라짐
   - 속도가 빠름

8. 보조 기억장치 (Secondary Storage)
   - Hard Disk Drive (HDD)
     - 물리적으로 펜이 돌아가면서 정보를 읽어옴
   - Solid State Drive (SSD)
     - USB가 발전된 형태
   - Floppy Disk Drive(FDD)
     - 내부 장치 펜이 돌아가면서 정보를 읽어옴 HDD랑 원리는 같음

- C드라이브가 기본 드라이브인 이유는 옛날에는 FDD가 A,B 드라이브 였기때문에

### 📌 **0.2 컴퓨터를 켤 때 일어나는 일들**

**부팅 절차**
1. 전원공급
2. 부트 프로그램 실행
3. 하드웨어 검사
4. 운영 체제 로드
5. 운영 체제 실행


- 메인보드
  - ROM BIOS
    - Read Only Momory(ROM)
    - 부트 프로그램 실행

- CPU 는 보조 기억장치에 접근을 못함 주 기억장치만 접근할 수 있음

### 📌 **0.3 운영체제가 해주는 일들**
- 운영 체제 Operating System (OS)

- unix
  - C언어는 에서 태어남

- Linux
  - 무료 os
  - 실무에서 많이 사용

**운영체제**
  - 커널 Kernel
    - 하드웨어의 입출력을 한번에 주고받기위해 운영체제가 만들어짐
    - 옛날에는 하드웨어 마다의 프로그램을 만들었었음
  - 자원을 관리해줌

### 📌 **0.4 컴퓨터는 왜 2진수를 사용할까**

- 컴퓨터는 모든 정보를 숫자로 바꾸어 받음
  - 이때 사용하는 숫자가 2진수

- 십진법 (Decimal numerical system)
- 이진법 (Binary numerical system)
  - Binary 는 0과 1 만사용하는 체계임

- 에디슨 효과 (리처디슨 효과, 열전자 방출)
  - 필라멘트에 열이 발생하면 전자가 방출됨
    - 이때 양극을 갖다대면 전자가 빨려들어감 이 운동하는 것을 전류라고함

  - 진공관의 원리
    - 전자가 음극에서 양극으로 가는것을 껏다 켯다 조절하는 것
      - 이렇게 껏다 켯다하는 것을 0과 1 로 표현됨

- 트랜지스터 (Transistors)
  - 반도체를 이용하며 만듬
    - ex) 실리콘 (Silcon)
  - 진공관의 기능과 유사함
    - 대신 진공관보다 크기가 작고 발열이없음
  - 콜렉터, 이미터, 베이스로 전류나 전압흐름을 조절하여 증폭하거나 스위치 역할을 하는 반도체 소자임

- 집적 회로 (Integrated Circuits)
  - 트랜지스터를 소형화하여 많이 집어 넣은 것

### 📌 **0.5 현대적 컴퓨터의 기본 구조**

**예전의 프로그래밍**

컴퓨터에 외부에있다가 갖다가 꽂아서 사용
  - 플로그보드에 전기배선 
  - 펀치 보드에 구멍을 뚫음
  - 애니그마

**엘런 튜링**
- 독일군에 훔쳐온 애니그마 암호를 해독함
- Universal Turing machine (UTM)
  - 입력으로 문자열 입력
  - 입력 문자열ㄹ 어떠한 기능을 수행함

**폰 노이만**
- 핵폭탄 개발시 컴퓨터 를 만듬
- 현대적 컴퓨터의 구조를 만듬
  - 내장형 프로그램 방식의 컴퓨터(Stored-program computer)
  - 폰 노이만 구조
    - 입력 장치가 cpu와 메모리 유닛의 연산을 통해 출력 장치로 내보내는 것

- 폰 노이만 구조가 발전하여 **Examle of a single system computer bus**
  - System bus
    - Control bus
      - CPU 와 메모리의 양방향 통신
    - Address bus
      - CPU의 입력
      - 메모리에 출력
    - Data bus
      - CPU 와 메모리의 양방향 통신


### 📌 **0.6 컴퓨터 메모리의 기본 구조**

- 연산은 CPU, 계산은 주기억장치-메모리가 함
- 캐시 메모리
  - 메모리에서 자주 사용하는 것을 캐쉬 메모리에 미리 저장해둠

- 순차 접근 (Sequential access)
  - 셀 하나씩 돌아가며 데이터를 찾는 방법

- 임의 접근 (Random access)
  - 특정 주소에 있는 데이터를 바로 찾아가는 방법

- RAM 의 구조
  - 트랜지스터 + 커패스터 를 합쳐서 하나의 셀을 이룸
    - 셀 - 1비트 씩 정보를 저장하는 공간 

- CPU ↔ 주 기억 장치의 통신 구조
  - 임의 접근을 하기 위해 Address bus 와 Data bus 를 구분함
  - Address bus 에서 저장되어있는 데이터를 찾음

### 📌 **0.7 CPU의 기본 구조**
- CPU의 언어
  - 기계어 ✔️
    - 2진수 언어
  - 어셈블리 언어 ❌
    - 기계어가 사람이 이해하기 쉽게 끔 바뀐 언어 **니모닉** 이라고함
  - 고수준 프로그래밍 언어 ❌

**CPU 명령어 집합**
- 어셈블리 언어의 기본 명령어
  - ADD - 숫자 두 개를 더함
  - COMPARE - 숫자들끼리 비교함
  - IN - 키보드 같은 입력장치로 부터 정보를 입력받음
  - JUMP - 지정된 메모리 주소로 점프함
  - JUMP IF - 조건에 따라 점프함
  - LOAD - 메모리에서 CPU로 정보를 가져옴
  - OUT - 출력 장치로 정보를 출력함
  - STORE - 메모리의 정보를 저장함

**CPU의 구성 요소들**

- 여러가지 레지스터들
  - 메모리와는 속도차이가 있음
  - 구조상으로는 메모리와 레지스터는 아주다르므로 서로 구분하는게좋음
- 산술 논리 장치(ALU)
  - Arithmetic Logic Unit
  - 실제 연산을 하는 곳
- 제어 장치

|레지스터 종류|기능|
|:--:|:--:|
|주소 레지스터|일거나 쓸 메모리 주소 저장|
|프로그램 카운터|다음 명령어의 메모리 주소 저장|
|데이터 레지스터|메모리에서 읽어온 데이터 저장|
|명령어 레지스터|메모리에서 읽어온 명령어 저장|
|어큐뮬레이터|연산에 사용되는 데이터 저장|

### 📌 **0.8 CPU가 일하는 방법**

1. **프로그램 카운터** 에 어셈블리 언어 명령어(EX) load 10)을 넣음
2. **메모리 주소 레지스터** 에 프로그램 카운터의 명령어를 넣음
3. **메모리 데이터 레지스터**에 메모리 주소 레지스터에 넣은 명령어의 주소에 접근하며 LOAD 10이라는 명령어를 가져옴 
4. **명령어 데이터 레지스터**가 명령어 레지스터에 LOAD10 이라는 명령어를 던져줌
5. **프로그램 카운터**가 올라감  
6. **제어장치**로 명령어 레지스터의 명령이 들어감
   - 이를 디코드 라고함 (명령 해독)
7. **메모리 주소 레지스터**에 10이라는 주소에 접근함
8. **메모리 데이터 레지스터**에 10이라는 주소 안에있는 3이라는 데이터를 가져옴  
9. **어큐물레이터** 로 3이라는 데이터가 감
10. **어큐뮬레이터**에서 산술논리장치로 데이터를 가져와서 연산하여 다시 **어큐뮬레이터**에 던져줌
11. 메모리 데이터 레지스터에 연산된 데이터를 던져줌 
- 이러한 방법이 반복되어 새로운 메모리에 계산된 데이터가 저장 됨
