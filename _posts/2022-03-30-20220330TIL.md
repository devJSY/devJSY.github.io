### 📌 **0.4 컴퓨터는 왜 2진수를 사용할까**

- 컴퓨터는 모든 정보를 숫자로 바꾸어 받음
  - 이때 사용하는 숫자가 2진수

- 십진법 (Decimal numerical system)
- 이진법 (Binary numerical system)
  - Binary 는 0과 1 만사용하는 체계임

- 에디슨 효과 (리처디슨 효과, 열전자 방출)
  - 필라멘트에 열이 발생하면 전자가 방출됨
    - 이때 양극을 갖다대면 전자가 빨려들어감 이 운동하는 것을 전류라고함

  - 진공관의 원리
    - 전자가 음극에서 양극으로 가는것을 껏다 켯다 조절하는 것
      - 이렇게 껏다 켯다하는 것을 0과 1 로 표현됨

- 트랜지스터 (Transistors)
  - 반도체를 이용하며 만듬
    - ex) 실리콘 (Silcon)
  - 진공관의 기능과 유사함
    - 대신 진공관보다 크기가 작고 발열이없음
  - 콜렉터, 이미터, 베이스로 전류나 전압흐름을 조절하여 증폭하거나 스위치 역할을 하는 반도체 소자임

- 집적 회로 (Integrated Circuits)
  - 트랜지스터를 소형화하여 많이 집어 넣은 것

### 📌 **0.6 컴퓨터 메모리의 기본 구조**

- 연산은 CPU, 계산은 주기억장치-메모리가 함
- 캐시 메모리
  - 메모리에서 자주 사용하는 것을 캐쉬 메모리에 미리 저장해둠

- 순차 접근 (Sequential access)
  - 셀 하나씩 돌아가며 데이터를 찾는 방법

- 임의 접근 (Random access)
  - 특정 주소에 있는 데이터를 바로 찾아가는 방법

- RAM 의 구조
  - 트랜지스터 + 커패스터 를 합쳐서 하나의 셀을 이룸
    - 셀 - 1비트 씩 정보를 저장하는 공간 

- CPU ↔ 주 기억 장치의 통신 구조
  - 임의 접근을 하기 위해 Address bus 와 Data bus 를 구분함
  - Address bus 에서 저장되어있는 데이터를 찾음

### 📌 **0.7 CPU의 기본 구조**
- CPU의 언어
  - 기계어 ✔️
    - 2진수 언어
  - 어셈블리 언어 ❌
    - 기계어가 사람이 이해하기 쉽게 끔 바뀐 언어 **니모닉** 이라고함
  - 고수준 프로그래밍 언어 ❌

**CPU 명령어 집합**
- 어셈블리 언어의 기본 명령어
  - ADD - 숫자 두 개를 더함
  - COMPARE - 숫자들끼리 비교함
  - IN - 키보드 같은 입력장치로 부터 정보를 입력받음
  - JUMP - 지정된 메모리 주소로 점프함
  - JUMP IF - 조건에 따라 점프함
  - LOAD - 메모리에서 CPU로 정보를 가져옴
  - OUT - 출력 장치로 정보를 출력함
  - STORE - 메모리의 정보를 저장함

**CPU의 구성 요소들**

- 여러가지 레지스터들
  - 메모리와는 속도차이가 있음
  - 구조상으로는 메모리와 레지스터는 아주다르므로 서로 구분하는게좋음
- 산술 논리 장치(ALU)
  - Arithmetic Logic Unit
  - 실제 연산을 하는 곳
- 제어 장치

|레지스터 종류|기능|
|:--:|:--:|
|주소 레지스터|일거나 쓸 메모리 주소 저장|
|프로그램 카운터|다음 명령어의 메모리 주소 저장|
|데이터 레지스터|메모리에서 읽어온 데이터 저장|
|명령어 레지스터|메모리에서 읽어온 명령어 저장|
|어큐뮬레이터|연산에 사용되는 데이터 저장|

### 📌 **0.8 CPU가 일하는 방법**

1. **프로그램 카운터** 에 어셈블리 언어 명령어(EX) load 10)을 넣음
2. **메모리 주소 레지스터** 에 프로그램 카운터의 명령어를 넣음
3. **메모리 데이터 레지스터**에 메모리 주소 레지스터에 넣은 명령어의 주소에 접근하며 LOAD 10이라는 명령어를 가져옴 
4. **명령어 데이터 레지스터**가 명령어 레지스터에 LOAD10 이라는 명령어를 던져줌
5. **프로그램 카운터**가 올라감  
6. **제어장치**로 명령어 레지스터의 명령이 들어감
   - 이를 디코드 라고함 (명령 해독)
7. **메모리 주소 레지스터**에 10이라는 주소에 접근함
8. **메모리 데이터 레지스터**에 10이라는 주소 안에있는 3이라는 데이터를 가져옴  
9. **어큐물레이터** 로 3이라는 데이터가 감
10. **어큐뮬레이터**에서 산술논리장치로 데이터를 가져와서 연산하여 다시 **어큐뮬레이터**에 던져줌
11. 메모리 데이터 레지스터에 연산된 데이터를 던져줌 
- 이러한 방법이 반복되어 새로운 메모리에 계산된 데이터가 저장 됨

### 📌 **0.10 2진수**

**10진수**
- Decimal numbers

**EX)**
1. 256
2. = 200 + 50 + 6
3. = 2 X 100 + 5 X 10 + 6 X 1
4. = 2 X 10(2제곱) + 5 X 10(1제곱) + 6 X 10(0제곱)

> **10(2제곱)** 이값을 BASE 라고하고 이 때문에 10진법이라고함

**부호가 있는 정수**
- Signed
- 2의 보수 방법
  - 숫자의 2진수 보수에 1을 더한값
  - 첫비트가 1이면 음수 0이면 양수
  - 2의 보수 방법의 장점은 -0과 0이 같음

- 부호가 있는 정수와 부호가 없는 정수 일때 계산 했을때 값이 달라짐

**8비트 정수의 범위**
- 부호가 있는 정수 (10진수 기준)
  - -128 ~ 127

- 부호가 없는 정수 (10진수 기준)
  - 0 ~ 255

>동일하게 256가지의 숫자 저장가능

### 📌 **1.3 C 언어의 표준**
- 사람이 만들었기에 시대에 따라서 표준이 달라짐

**K&R C(Classic C)**
- 유닉스와 함께 제동된 라이브러리가 사실상의 표준
- 1978

**The Spirit of C**
- 1988
- 프로그래머를 믿어라
- 프로그래머가 필요한 일을 하는 것을 방해하지 마라
- 언어를 작고 단순하게 유지해라
- 한 연산을 할 떄 단 한 가지 방법만 제공해라
- 호환을 보장할 수 없다 하더라도 속도를 빠르게 해라

**ANSI/ISO C9X Committee**
- 국제적인 문자 집합 추가
- 64비트 프로세서 대응
- 과학 공학 수치 계싼 적합도 개선
- 언어를 단순하게 유지
- 커미티는 C++ 가 크고 야심한 언어가 되는 것을 허용하는 것에 동의했다

**The C11 Standard**
- C99의 일부를 선택적인 기능으로 변경
- 멀티 프로세서 대응

**The C18 Standard**
- 새로운 기능 추가 없이 C11의 결함 수정

### 📌 **1.11 명령 프롬프트의 기본적인 사용법**

- `explorer .`
  - explorer 폴더위치를 열어줌
  - .은 현재위치 를 의미함

- `cd`
  - change Directory 라는 뜻
  - cd .. 누르면 상위 디렉토리로 이동
  - cd 누르고 Tab 키누르면 디렉토리들을 찾아줌
    - Shift + Tab 누르면 이전 디렉토리를 찾아줌
  - cd 치고 한칸띄고 폴더를 잡아끌어다 놓으면 그 폴더의 위치로 이동함
  - cd치고 폴더 경로를 넣으면 그 폴더의 위치를 이동함

- `dir`
  - 현재 접속된 하위 디렉토리를 보여줌

- 빌게이츠가 마이크로 소프트를 만들고 MSDos 를 만들때 부터 사용한 명령어임

- `d:`
  - d드라이브로 이동

### 📌 **1.13 비주얼 스튜디오 코드의 기본적인 사용 방법**

- F1 키로 설정검색가능
- vsc에서는 gcc랑 mingw-w64 다운로드해야 컴파일 할 수 있음
- vsc에서 C언어 컴파일 방법
  1. powershell에 gcc 엔터
  2. main.c 엔터 
  3. 실행코드 생성됨
  4. .\a.exe 실행하면 실행됨

- vsc에서 c++ 컴파일 방법
  1. powershell에 g++ .\main.cpp 엔터
  2. a.exe 생성됨
  3. .\ a.exe 실행

- .누르고 / 누르고 탭누르고 \ 으로 자동완성됨
- gcc에서 오브젝트 파일을 보고싶다면 gcc -c main.c 
  - 오브젝트파일만 생시고 컴파일 파일이 안생김
